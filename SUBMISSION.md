## Problem & Solution

### The problem

Onboarding a new engineer at Webflow costs a manager somewhere between
four hours and four days of spreadsheet wrangling. Managers have to:

- Remember every team-specific ritual, tool, and doc link.
- Pull the right people across engineering, product, design, and
  leadership for a first-month meet list.
- Write a warm, personal welcome note the hire will actually read.
- Repeat the same work every time they hire.

The new hire then spends their first days hunting down the same
context — in Confluence, in Slack DMs with their manager, and in
whatever ad-hoc channels their team prefers. The handoff is static,
manager-owned, and disconnected from the Slack experience the hire
lives in from day one.

### The solution — three agents, one edge deployment

Spark is a multi-agent system that turns onboarding into a reviewable
draft in minutes:

1. **Generator agent** — Anthropic tool-use loop with six real tools.
   It calls `resolve_new_hire` against Slack, `fetch_team_roster` to
   pull actual engineer peers / PM / designer / director / people
   partner via custom Slack profile fields and manager-chain
   traversal, `find_stakeholders` to look up Confluence user guides,
   `draft_welcome_note` to stream two welcome voices (manager + Spark)
   the instant they're written, `tune_checklist` to add team-specific
   items, and `finalize_draft` to commit the full plan. Up to 20
   iterations with per-tool timeouts and schema validation.
2. **Critique agent** — reads the finalized draft and returns
   structured findings (missing buddy, thin welcome note, roster
   mismatches). Each finding ships with a one-click patch.
3. **Slack assistant agent** — once the manager publishes, Spark
   delivers a Slack canvas + DM + Home tab to the new hire, and the
   assistant-thread agent takes over from there as a live ramp-up
   companion.

Per-person **"Ask me about..." blurbs** are generated by a separate
pipeline that pulls each teammate's recent Jira tickets (via
`/rest/api/3/search/jql`) and recent GitHub pull requests, then feeds
that real activity to Claude for a warm, specific one-liner. No
fabricated teammates — every person carries a verified Slack ID.

The manager is always in the loop. Spark hands over a draft, surfaces
candidates (including buddy suggestions), and waits for the manager to
assign the buddy, edit the welcome, tweak the checklist, and
explicitly hit publish.

Everything runs inside a single Next.js app on a single Cloudflare
Worker — manager UI, Slack webhooks, D1 draft store, all three agents,
entire system on the edge.

### Why this fits "Agents of Possibility"

- **LLM as reasoning engine, not autocomplete.** The generator
  _decides_ which tool to call next based on what it learned from the
  previous call. That's visible live in the streaming agent timeline.
- **Real tool use, real systems.** Slack's API, Jira's JQL search,
  GitHub's issues search, Confluence, and a D1 draft store — the
  agent mutates production state (Slack canvases, channels, DMs), not
  just a sandbox.
- **Multi-agent collaboration.** Generator produces the draft,
  Critique reviews it, Slack assistant delivers it. Different jobs,
  one shared runtime.
- **Autonomous but supervised.** Multi-step loop with checkpoints;
  nothing ships until the manager clicks publish.
- **An actual AI teammate.** A manager with Spark onboards a new hire
  in minutes instead of days. The checklist is team-tuned. The people
  to meet are real. The welcome has voice. That's the "teammate"
  promise.

## Setup instructions

See [`README.md`](README.md#quick-start) for the full quick-start.
The short version:

```bash
git clone https://github.com/javonmcgilberry/spark.git
cd spark
npm install
npm run dev
```

Zero tokens required — Slack and Anthropic fall back to mock clients
if `.env` is empty.

For the real Worker + D1 runtime locally:

```bash
npm run preview
```

For real Slack round-trips (Cloudflare Tunnel + dev Slack app), see
[`docs/dev-setup.md`](docs/dev-setup.md).

## Required environment variables

Full list with comments in [`.env.example`](.env.example).

```env
# Slack
SLACK_BOT_TOKEN=xoxb-...
SLACK_SIGNING_SECRET=...

# Anthropic
ANTHROPIC_API_KEY=sk-ant-...
ANTHROPIC_MODEL=claude-3-5-haiku-latest

# Integrations (optional — missing values just disable those tools)
GITHUB_TOKEN=ghp_...
JIRA_BASE_URL=https://webflow.atlassian.net
JIRA_API_TOKEN=...
CONFLUENCE_BASE_URL=https://webflow.atlassian.net/wiki
CONFLUENCE_API_TOKEN=...

# Optional: Atlassian OAuth (per-viewer tokens, preferred over Basic)
ATLASSIAN_OAUTH_CLIENT_ID=...
ATLASSIAN_OAUTH_CLIENT_SECRET=...

# Local-dev session fallback (on Webflow Cloud this comes from CF Access)
DEMO_MANAGER_SLACK_ID=U...
DEMO_MANAGER_EMAIL=you@webflow.com
```

D1 is declared in `wrangler.jsonc` and Webflow Cloud auto-provisions it
at deploy time — no env var needed.

## Demo steps

See [`docs/manager-dashboard-demo.md`](docs/manager-dashboard-demo.md)
for the full runbook. The ~4-minute happy path:

1. Open the dashboard.
2. Click **Create onboarding plan** → pick a new hire from the Slack
   picker.
3. Click **Create draft & run agent** → watch the generator timeline
   stream live (`resolve_new_hire → fetch_team_roster →
draft_welcome_note → find_stakeholders → tune_checklist →
finalize_draft`). The welcome note persists the instant
   `draft_welcome_note` fires.
4. In the draft workspace: assign the buddy (manager's call — Spark
   never auto-selects), edit anything, click **Ask agent to review**
   to run the Critique agent.
5. Apply any finding with one click.
6. Click **Publish to Slack**. The new hire receives a Slack canvas +
   DM + a Home tab, and the Slack assistant agent takes over.
